# MVP 实现计划

## 阶段 1：数据底座

### 目标

通过 Parser Skill / 离线工具完成民法典 docx 解析和入库。

### 任务

- 建表 `law_documents`。
- 建表 `law_articles`。
- 建表 `knowledge_import_jobs`。
- 实现 Parser Skill 原型。
- 实现 docx 读取。
- 实现条文解析。
- 输出标准结构化 JSON / SQL。
- 写入 `law_documents` 和 `law_articles`。
- 记录导入任务。

### 验收

- `中华人民共和国民法典_20200528.docx` 可读取。
- 能识别标题：中华人民共和国民法典。
- 能识别通过日期：2020-05-28。
- 能解析出 1260 条。
- 每条有条号、正文、章节路径。
- Spring Boot 后端不直接承担文件解析职责。

## 阶段 2：关键词检索

### 目标

不用 AI，先能查法条。

### 任务

- 实现法规名称搜索。
- 实现条号搜索。
- 实现正文关键词搜索。
- Spring Boot 消费 `law_documents` 和 `law_articles`。
- 按 DDD 分层实现 `law` 上下文。
- 实现 `SearchLawArticlesUseCase`。
- 实现 `LawArticleRepository` 端口和 PostgreSQL 适配器。
- 实现 `GET /api/law/articles/search`。
- 实现法规库页。
- 实现专业检索页基础版本。

### 验收

- 搜索 `第五百七十七条` 能精确命中。
- 搜索 `违约责任` 能返回相关条文。
- 搜索 `借款合同` 能返回对应章节条文。

## 阶段 3：RAG 检索

### 目标

加入 embedding 和 hybrid search。

### 任务

- 开启 pgvector。
- 对每条 `law_articles.content` 生成 embedding。
- 写入向量索引。
- 实现 `HybridSearchService`。
- 实现 RAG 检索测试页。

### 验收

- 输入 `别人欠钱不还怎么办`，能命中借款合同、违约责任、诉讼时效相关条文。
- 检索测试页能显示关键词命中、向量命中和 rerank 后结果。
- 每条结果显示命中方式和相关度。

## 阶段 4：Agent 问答

### 目标

基于检索结果生成带引用回答，并通过 AG-UI 协议对外提供流式对话能力。

### 任务

- 接入 AgentScope Java v2。
- 实现 AG-UI 接口。
- 实现公众咨询页。
- 实现右侧引用依据面板。
- 将现有 `law/rag` use case 提升为 `searchLawArticles`、`getArticleDetail`、`validateCitations` tools。
- 将 `citations`、`answerSegments`、会话状态接入 AG-UI。

### 验收

- 用户问 `别人欠钱不还怎么办`。
- 系统先检索相关法条。
- 回答中有依据标签。
- 右侧显示引用法条。
- 引用能回到原文条文。
- 无依据时不编造。
- 前端通过 AG-UI 流式拿到回答过程与最终状态。

## 阶段 5：专业检索增强

### 目标

让内部专业用户能整理和复用法律依据。

### 任务

- 加入依据清单。
- 复制法律依据。
- 导出 Markdown。
- 生成专业法律依据段落。

### 验收

- 用户搜索 `违约责任`。
- 可选择 3 条依据加入清单。
- 可生成一段法律依据文字。
- 所有引用都能回到原文。

## 阶段 6：评测与质量控制

### 目标

防止检索和 Agent 回答效果退化。

### 任务

- 建立 10 个 MVP 测试问题。
- 实现评测集。
- 实现运行评测。
- 实现人工评分。
- 检查引用真实性。

### 初始评测问题

- 公司拖欠工资怎么办？
- 房东不退押金怎么办？
- 别人欠钱不还怎么办？
- 网购商品质量问题可以退吗？
- 被公司辞退有没有补偿？
- 整理劳动合同解除相关法律依据。
- 查询民法典中关于违约责任的规定。
- 帮我找消费者退货退款相关法律依据。
- 这段合同里的违约金条款涉及哪些法律规定？
- 生成一份欠款纠纷的法律依据清单。

### 验收

- 每个问题记录检索结果、回答、引用和人工评分。
- 引用真实性必须 100%。
- 不允许出现虚构法条。

## 第一周建议目标

第一周只完成前三阶段的一部分：

```text
上传民法典
 -> 解析 1260 条
 -> 入库
 -> 关键词搜索
 -> RAG 检索测试雏形
```

数据链路稳定后，再接 Agent 和前端流式体验。
